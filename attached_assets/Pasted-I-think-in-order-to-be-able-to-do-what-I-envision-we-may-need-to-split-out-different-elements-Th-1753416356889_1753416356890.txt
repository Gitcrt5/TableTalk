The main issue is that currently there is no single point of truth for the definition of a game. What one user specifies for the name and location may be different from what nother user will use. This could cause issues when one person uploads the PBN file and others search for or use the game. It could also cause conflicts if more than 1 player tries to upload a PBN file.

In the future I can see that clubs may upload game details with PBN files that players can use. Until then we will need to use user supplied data with inherrent inconsistencies that we do our best to mitigate. I can supply a list of bridge clubs. Players could have a "Home" club which would be the default location when there needs to be a choice. We could also have Favourite clubs which would be a short list to choose from, and then a full list available. When players specify a location they need to choose from this list. There would be a choice "Other" if one is not specified.

The possibility of duplicate PBN files should be allowed for given the above. When a user uploads a PBN it could check against existing PBN files. How this is done should balance resource (CPU and database) utilisation against effeciency. This may not be a feasible option long term. If a duplicate is found it could present the games it found and ask if one of those games should be used. If yes then use that game, otherwise then prompt for the name, date, location.
